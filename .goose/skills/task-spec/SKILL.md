# Task: create spec

## When to use
- The user asks for to start working on a task
- You need more context before working on a task

## Procedure

**as a Project Manager**
Review the work plan documentation, check ./Plan.md. Prepare a detailed task specification from project owner POV. Write it to .specs/task-<id>.md

**as an Architect**
Review the work plan documentation, check ./Plan.md. Update the task specification with technical architecture details. Write it to .specs/task-<id>.md

**as QA Analyst**
Review the work plan documentation, check ./Plan.md. Update the task specification with requited contract details from QA perspective. List required tests and acceptance criteria. Write it to .specs/task-<id>.md
